# FTW Agency Website

A premium, dark-themed growth platform for FTW Agency — positioning the company as a **growth systems company**, not a traditional marketing agency. Built on Next.js 15 with a typed local-content CMS layer architected to scale to 500+ programmatic SEO pages.

## Stack

- **Next.js 15** (App Router, fully static generation — 173 pages)
- **TypeScript** end-to-end
- **Tailwind CSS v4** (token-driven theme in `app/globals.css`)
- **Framer Motion** for scroll reveals, the pinned Growth Framework, and micro-interactions
- **Zod** for build-time content validation
- **Typography:** General Sans (self-hosted) for headings, Instrument Serif italic for headline accents, Inter for body

## Getting started

```bash
npm install
npm run dev      # development server
npm run build    # production build (static)
npm run start    # serve the production build
```

## Architecture

### Content layer (the "CMS")

All content lives in `/content` as typed files validated by Zod schemas (`lib/schemas.ts`):

| Collection | Path | Entries |
|---|---|---|
| Services | `content/services/*.ts` | 10 |
| Industries | `content/industries/*.ts` | 10 |
| Locations | `content/locations/*.ts` | 3 (template scales to hundreds) |
| Portfolio | `content/portfolio/*.ts` | 6 |
| Case studies | `content/case-studies/*.ts` | 4 |
| Blog | `content/blog/*.mdx` | 6 |
| Site config | `content/site.ts` | nav, testimonials, FAQs, stats |

**Adding a page = adding a content file.** New service, industry, or location entries automatically generate their detail pages, combo pages, sitemap entries, nav/footer links, and command-palette results.

The access layer (`lib/collections.ts` + `lib/content.ts`) is the only code that touches content sources — swapping to a headless CMS later means rewriting those two modules, never the templates.

### Programmatic SEO

- **Combo routes:** `/industries/[industry]/[service]`, `/locations/[location]/[service]`, `/locations/[location]/industries/[industry]` — copy generated by the merge functions in `lib/combo.ts`
- **Metadata factory:** `buildMetadata()` in `lib/metadata.ts` (title/description/OG/canonical)
- **Structured data:** `lib/jsonld.ts` — Organization, LocalBusiness, Service, FAQPage, BreadcrumbList, Article
- **Dynamic sitemap:** `app/sitemap.ts` derives every URL from the collections
- **Internal linking:** `components/blocks/Related.tsx` ensures no page is an orphan

### The Growth Engine

The signature visual (`components/hero/GrowthEngineCanvas.tsx`) is a canvas-rendered network — growth inputs → growth assets → core → outcomes — with labeled mini-UI chips, data-flow particles, cursor interactivity, and a staged entrance. The same component is reused in the scroll-pinned Growth Framework section, rewiring itself per stage (Attract → Convert → Automate → Scale).

The hero's **industry switcher** (`IndustryContext`) live-swaps floating panels, ticker events, and example metrics, and highlights the selected industry across the homepage.

### Performance & accessibility

- Fully static output, zero server dependencies
- Canvas pauses off-screen via IntersectionObserver; particle counts adapt to viewport
- Global `prefers-reduced-motion` support (static network render, no animations)
- `Cmd+K` command palette over every content collection

## Placeholder content

Screenshots, portfolio imagery, and blog covers render as stylized mock panels (`components/ui/MockPanel.tsx`) until real assets are uploaded — the schemas already carry the fields. Contact form submits to a local success state; wire `components/contact/StrategyCallForm.tsx` to your CRM/scheduler for production.
